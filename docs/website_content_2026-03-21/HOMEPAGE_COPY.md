## Section: What It Is

CerbiBot is built for people who want more control than a chat app and less chaos than an unstructured agent stack.

It combines:
- a local daemon API
- a dashboard for operators
- CLI control for technical workflows
- governed execution for skills and risky actions

## Section: Why It Exists

Most AI tools optimize for quick conversation. CerbiBot is built for repeatable work:
- structured orchestration modes
- visible runs, checkpoints, heartbeats, and blockers
- reusable skills and artifacts
- project isolation and operator control

## Section: Core Features

### Multi-Mode AI Work
Choose the right execution style for the task:
- `single` for direct answers
- `critique` for review/refinement
- `retrieval` for grounded responses
- `debate`, `consensus`, and `council` for structured multi-pass reasoning

### Governed Skills
Run workflow skills with stronger controls:
- manifest requirements
- validation and testing
- approval-aware execution
- signing and verification support

### Operator Dashboard
Monitor and manage the system from one interface:
- chat and sessions
- runs and stalled-heartbeat visibility
- settings and provider routing
- skills, memory, and artifacts

### Project Isolation
Keep work separated by project:
- project-scoped sessions
- project-scoped memory
- cleaner workflow boundaries

### Provider and MCP Control
Operate models and tool servers more deliberately:
- provider configuration and testing
- model catalog and model discovery
- MCP server configuration and health checks

### Delegation and Artifacts
Support repeatable execution:
- patch-first delegation workflow
- artifact capture and export
- webhook-triggered runs

## Section: Security and Control

CerbiBot is built with operator control in mind:
- bearer-token protected API
- admin-password gates for sensitive actions
- audit logging
- export authorization checks
- encryption-at-rest support

## Section: Who It Is For

- builders who want repeatable orchestration instead of one-off prompting
- teams who need visibility into AI execution state
- users who care about governance, auditability, and local control
