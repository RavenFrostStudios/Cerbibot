# Launch Announcement Draft

Updated: 2026-03-21

## Short Version

CerbiBot is now available as a public technical preview.

It is a self-hosted orchestration platform for local AI workflows with multi-mode execution, governed skills, project-scoped memory, provider controls, artifacts, audit logging, and a dashboard built for operator visibility.

This preview is for users who want more control than a plain chat interface and more structure than an ad hoc agent stack.

## Longer Version

We are opening CerbiBot as a public technical preview.

CerbiBot is designed as a local, security-first AI control plane for people running serious workflows, not just one-off prompts. It combines:
- multi-mode orchestration
- governed skill execution
- provider and MCP controls
- project-scoped sessions and memory
- artifacts, audit logging, and admin-gated export paths
- a dashboard and CLI for direct operator control

Current strengths:
- `single`, `critique`, `retrieval`, `debate`, `consensus`, and `council` modes
- skill validation, testing, signing, and verification support
- run tracking with checkpoints, heartbeats, blockers, and recovery-aware controls
- project isolation and artifact handling

Current preview scope:
- local/self-hosted operator workflows
- dashboard + CLI usage
- provider-backed orchestration with governance controls

Still expanding:
- broader connector coverage
- more onboarding polish
- additional public docs and packaging improvements
